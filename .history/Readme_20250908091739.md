# Compliance Frontend

Este repositório contém a aplicação frontend do Sistema de Compliance, desenvolvido em Angular.

## Estrutura do Projeto

```
compliance-frontend/
  ├── compliance-web/     # Código-fonte da aplicação Angular
  ├── docker-compose.yml  # Configuração para rodar o frontend isoladamente
  └── Readme.md           # Este arquivo
```

## Pré-requisitos

- Docker e Docker Compose
- Node.js e npm (para desenvolvimento local)

## Configuração e Execução

### Usando Docker Compose

Para executar o frontend isoladamente:

```bash
docker-compose up
```

Isso iniciará o container do frontend na porta 4200. Acesse a aplicação em: http://localhost:4200

### Conectando ao Backend

Por padrão, o frontend tentará se conectar ao backend através do endereço `http://backend-api:8080/api`. 

Para conectar a um backend em execução em outro lugar:

1. Edite o arquivo `proxy.conf.json` dentro da pasta `compliance-web`:
   ```json
   {
     "/api": {
       "target": "http://seu-backend-url:8080",
       "secure": false,
       "logLevel": "debug",
       "changeOrigin": true
     }
   }
   ```

2. Reconstrua e reinicie o container:
   ```bash
   docker-compose down
   docker-compose up --build
   ```

## Desenvolvimento Local

Para desenvolvimento sem Docker:

1. Entre na pasta da aplicação:
   ```bash
   cd compliance-web
   ```

2. Instale as dependências:
   ```bash
   npm install
   ```

3. Inicie o servidor de desenvolvimento:
   ```bash
   ng serve --proxy-config proxy.conf.json
   ```

A aplicação estará disponível em http://localhost:4200.

## Integração com o Backend

Este frontend está projetado para funcionar com o backend disponível no repositório separado. Certifique-se de que o backend esteja em execução e acessível conforme configurado no arquivo `proxy.conf.json`.
